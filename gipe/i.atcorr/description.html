DESCRIPTION

i.atcorr does atmospheric corrections on an input raster. A detailed description can be obtained at http://www.ltid.inpe.br/dsr/mauro/6s/.

Flags:

-f
Force data to be output as FCELL or DCELL values. Default is CELL. Use this if output scale is chosen to be [0,1].
-r
Input data converted to reflectance, otherwise data will be treated as radiance values and will first be converted to reflectance before it is transformed. The output data is always reflectance values.
-a
Input raster is from an Landsat etm+ satellite image taken after July 1, 2000. The necesary conversions will be made.
-b
Input raster is from an etm+ satelite image taken before July 1, 2000. The necesary conversions will be made.
-o
Because using a continues elevation or visibility map makes execution time much longer, it is advised to use categorized maps instead. This options tells i.atcorr to try and speedup calculations. It is applicable if only one of the two kinds are maps are used and not the other. NOTE: this option under certain conditions can make execution time longer.

Parameters:

iimg=name
Name of the input raster to do the atmospheric correction on. The input raster should contain values with in the range specified.
icnd=name
Name of the ASCII file that contains input conditions used in the calculation.
Input conditions file format are explained at the end of this document (After AUTHORS).
oimg=name
Output raster file name. The generated output raster will be scaled to the values specified by the oscl field.
ialt=name
Name of a raster that contains elevation data to be used in the calculation. This will cause the calculation to take significantly longer. The elevation values should be in km.
ivis=name
Name of a raster that contains visibility data to be used in the calculation. This will cause the calculation to take significantly longer. The visibility values should be in km.
iscl=input scale range
Range to which data is scaled. By default this range is [0,255] since data used to be read from .RAW files containing unsigned chars.
oscl=output scale range
Range to which output data will be scaled. The default is [0,255] to match tihs input default.

NOTES

i.atcorr ignores the current region setting and adjusts the region to the whole image before the atmospheric corrections are made. This should not affect the user's region setting. Here is an example 6s file that runs this GRASS GIS version:
8 (Landsat ETM+ geometrical conditions)
2 19 13.00 -47.410 -20.234 (month,day,hh.ddd,long.,lat.) (hh.ddd=the decimal hour in universal time)
1 (tropical atmospheric mode)
1 (continental aerosols model)
15 (visibility in km (aerosol model concentration)
-.600 (target at 600 m above sea level)
-1000 (sensor on board of satellite)
64 (fourth band of ETM+ Landsat 7)

D. Aerosol model concentration: visibility

if you have an estimate of the meteorological parameter visibility v, enter directly the value of v in km (the aerosol optical depth will be computed from a standard aerosol profile)

if you have an estimate of aerosol optical depth, enter v=0 for the visibility and enter the aerosol  optical depth at 550

warning:  if iaer=0, enter v=-1



E. Target alitude sensor platform:

xps <=0. means the target is at the sea level
xps >0. means you know the altitude of the target expressed in km and you put that value as xps

xpp is the parameter to express the sensor altitude
         xpp= -1000  means that the sensor is a board a satellite
         xpp=     0  means that the sensor is at the ground level

for aircraft simulations  (xpp is neither 0 nor -1000)

-100< xpp <0  means you know the altitude of the sensor expressed  in kilometers; this altitude is relative to the target altitude
for aircraft simulations only, you have to give
puw,po3   (water vapor content,ozone content between the   aircraft and the surface)
taerp     (the aerosol optical thickness at 550nm between the aircraft and the surface)
if these data are not available, enter negative values for all of them, puw,po3 will then be interpolated from the us62 standard profile according to the values at ground level. Taerp will be computed according to a 2km exponential profile for aerosol.
